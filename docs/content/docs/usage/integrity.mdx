mpm は、プラグインをダウンロードした際にファイルの整合性（ハッシュ）を検証します。これにより、破損・切り詰め・改竄されたファイルや、エラーページが誤ってプラグインとして保存されることを防ぎます。npm や cargo、apt といった成熟したパッケージマネージャと同様に、インストール時のハッシュ検証を標準で行います。

## 🧭 検証のしくみ

プラグインをインストール・更新すると、mpm はダウンロードした一時ファイルを、サーバーの `plugins/` へ配置する前に検証します。照合には次の優先順位でハッシュを使用します。

1. **リポジトリが提供するハッシュ** — Modrinth は sha1 / sha512、Hangar は sha256 を API で提供します。これらが取得できる場合は、実際にダウンロードするファイルのハッシュと照合します。
2. **保存済みの sha256** — 同じバージョンを再取得する場合に限り、過去のインストール時に記録した sha256 と照合します。これは GitHub や SpigotMC のようにハッシュを提供しないリポジトリでも、同一バージョンの再インストールを自己検証できるようにする仕組み（trust-on-first-use）です。バージョンが変わる更新では、旧バージョンのハッシュは照合に使用しません。
3. **照合先がない場合** — 検証は行えませんが、計算した sha256 を記録し、次回以降の検証に備えます。

検証に成功すると、計算した sha256 がプラグインのメタデータ（`plugins/mpm/metadata/<plugin>.yaml` の `mpmInfo.download.sha256`）へ保存されます。

<Callout type="warn" title="ハッシュが一致しない場合">
ダウンロードしたファイルのハッシュが期待値と一致しない場合、インストールは中断され、一時ファイルは削除されます。ファイルが破損しているか、リポジトリ側で差し替えられた可能性があります。
</Callout>

## 🚑 検証をスキップする

正当な理由でハッシュの不一致を無視したい場合は、`--skip-integrity` フラグを指定します。このフラグは `add` / `install` / `update` の各コマンドで利用できます。

```bash
mpm install --skip-integrity
mpm add QuickShop --skip-integrity
mpm update --skip-integrity
```

<Callout type="info" title="force との違い">
`--force` は api-version の非互換を無視するためのフラグで、整合性検証には影響しません。整合性の不一致を無視するには、必ず `--skip-integrity` を使用してください。両者は独立したフラグです。
</Callout>

<Callout type="warn" title="スキップ時のハッシュは記録されません">
`--skip-integrity` で不一致を無視してインストールした場合、そのファイルは検証をパスしていないため、sha256 は「検証済み」として記録されません。そのため後続の `mpm verify` は、そのプラグインを「ハッシュ未保存で検証不可」として扱い、誤って「検証OK」とは報告しません。
</Callout>

## ✅ インストール済みプラグインを再検証する

`mpm verify` を実行すると、インストール済みの管理下プラグインの JAR を、メタデータに保存された sha256 と照合します。このコマンドはネットワークにアクセスせず、ローカルの検証のみを行います。

```bash
mpm verify
```

検証結果は次の状態で表示されます。

- **検証OK** — 保存済みハッシュと一致しています。
- **ハッシュ不一致** — JAR が保存時から変化しています。破損や改竄が疑われます。`mpm update <plugin>` で再取得してください。
- **JARファイルが見つかりません** — メタデータに記録されたファイルが存在しません。
- **ハッシュ未保存で検証不可** — sha256 が記録されていないため検証できません。一度インストールまたは更新を行うと記録されます。
