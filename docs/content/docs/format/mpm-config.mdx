## 概要

`mpm.json`は、mpmのプロジェクト設定ファイルです。
npmの`package.json`に相当し、プロジェクトで使用するプラグインの依存関係を宣言的に管理します。

<Callout type="info" title="設定ファイルの分離">
mpmの設定は、**プロジェクト設定** と **プラグイン設定** の2つに分離されています。

- [`mpm.json`](/docs/format/mpm-config)（このファイル）— プロジェクトの名前・バージョン・プラグイン依存関係を宣言（チーム共有・バージョン管理対象）
- [`config.json`](/docs/format/config) — リポジトリソース・グローバル設定・Discord Webhook・スケジュール自動更新（サーバー固有）

リポジトリソースやWebhook、スケジュール設定は `config.json` 側に記述します。
</Callout>

## ファイルの配置

```
your-server/
├── mpm.json          ← このファイル（プロジェクト設定）
├── mpm-lock.yaml     ← 自動生成されるロックファイル
├── plugins/
│   └── mpm/
│       ├── config.json  ← サーバー固有のmpm設定
│       └── metadata/    ← 個別プラグインの詳細情報
└── ...
```

## 基本構造

```jsonc title="mpm.json"
{
    "name": "main-server",               // プロジェクト名
    "version": "1.0.0",                  // プロジェクトのバージョン
    "plugins": {
        "LuckPerms": "5.4.97",          // プラグイン依存関係
        "Vault": "unmanaged"
    }
}
```

## フィールド詳細

### name（プロジェクト名）

プロジェクトの識別名を指定します。必須項目です。

```json
{
    "name": "main-server"
}
```

### version（プロジェクトバージョン）

プロジェクト自体のバージョンを指定します。

```json
{
    "version": "1.0.0"
}
```

- **デフォルト値:** `"1.0.0"`
- プラグインのバージョンではなく、プロジェクト（mpm.json）のバージョンを指す

### plugins（プラグイン依存関係）

プロジェクトで使用するプラグインを定義します。

```json
{
    "plugins": {
        "LuckPerms": "5.4.97",
        "Vault": "latest",
        "CarbonChat": "tag:beta",
        "EssentialsX": "unmanaged",
        "QuickShop-Hikari-Addon": "sync:QuickShop-Hikari"
    }
}
```

- **キー**: プラグイン名（リポジトリで定義されているプラグインID）
- **値**: バージョン指定文字列

#### バージョン指定形式

| 形式 | 説明 | 例 |
|------|------|-----|
| `"X.Y.Z"` | 固定バージョン | `"5.4.97"` |
| `"latest"` | 最新バージョン | `"latest"` |
| `"tag:チャンネル"` | リリースチャンネル指定 | `"tag:beta"` |
| `"sync:PluginName"` | 別プラグインと同期 | `"sync:QuickShop"` |
| `"unmanaged"` | 手動管理 | `"unmanaged"` |

#### 🏷️ tag指定（リリースチャンネル）

`tag:チャンネル名` 形式を使用すると、特定のリリースチャンネル（安定版、ベータ版など）の最新バージョンを自動的に取得できます。

```json
{
    "plugins": {
        "CarbonChat": "tag:beta",
        "LuckPerms": "tag:release"
    }
}
```

**サポートされるチャンネル:**

| チャンネル | 説明 |
|-----------|------|
| `release` | 安定版リリースのみ |
| `beta` | ベータ版リリース |
| `alpha` | アルファ版リリース |

**プラットフォーム別の対応状況:**

| プラットフォーム | 対応状況 | 備考 |
|----------------|---------|------|
| Modrinth | ✅ 対応 | `version_type` フィールドでフィルタリング。`release`/`beta`/`alpha` を区別可能 |
| GitHub | ✅ 対応 | `prerelease` フラグでフィルタリング。`beta` と `alpha` は区別不可（どちらも prerelease 扱い） |
| SpigotMC | ❌ 非対応 | チャンネルの概念がないため、`tag:` 指定時はエラーになります |

<Callout type="warn" title="GitHubでの制限">
GitHubは `prerelease` の2値（true/false）しか持たないため、`tag:beta` と `tag:alpha` は同じ結果を返します。
ベータ版とアルファ版を明確に区別したい場合は、Modrinthの使用を推奨します。
</Callout>

<Callout type="info" title="リポジトリ側でのチャンネル既定値">
プラグインによってはstableではなくbeta等を通常運用している場合があります。その場合は
`mpm.json` 側で毎回 `tag:beta` を書くのではなく、[リポジトリファイル](/docs/format/repository)の
`defaultVersion` フィールドでチャンネル既定値を指定するのが推奨です。
`"latest"` と書いたときに自動的に `defaultVersion` が解決されます。
</Callout>

#### 🔗 sync指定（バージョン同期）

`sync:PluginName` 形式を使用すると、アドオンプラグインを親プラグインのバージョンに同期できます。

```json
{
    "plugins": {
        "QuickShop-Hikari": "6.2.0",
        "QuickShop-Hikari-Addon": "sync:QuickShop-Hikari"
    }
}
```

**動作:**
- インストール時: `QuickShop-Hikari-Addon` は `QuickShop-Hikari` と同じ `6.2.0` をインストール
- 更新時: `QuickShop-Hikari` が更新されると、アドオンも連動して更新

**制限事項:**
- 同期対象プラグインは `mpm.json` に存在する必要があります
- 同期の連鎖（sync:A → sync:B）はサポートされていません
- 循環依存（A → B → A）は検出されエラーになります
- `sync:` 指定のプラグインは [`mpm pin` / `mpm pin-all`](/docs/commands/manage) で固定できません（固定すると同期関係が壊れるため、対象から除外されます）

<Callout type="info" title="ユースケース">
QuickShopやEssentialsXなど、多くのアドオンを持つプラグインで特に便利です。
親プラグインのバージョンを変更するだけで、関連するすべてのアドオンが自動的に同じバージョンに更新されます。
</Callout>

## 使用例

### シンプルな構成

基本的な依存関係宣言：

```json
{
    "name": "my-server",
    "version": "1.0.0",
    "plugins": {
        "LuckPerms": "latest",
        "Vault": "latest"
    }
}
```

### 複数の指定形式を組み合わせた構成

```json
{
    "name": "production-server",
    "version": "2.0.0",
    "plugins": {
        "LuckPerms": "5.4.97",
        "PlaceholderAPI": "latest",
        "CarbonChat": "tag:beta",
        "QuickShop-Hikari": "6.2.0",
        "QuickShop-Hikari-Addon": "sync:QuickShop-Hikari",
        "EssentialsX": "unmanaged"
    }
}
```
