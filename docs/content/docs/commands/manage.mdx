## ワークフロー

プラグイン管理の基本的なワークフローです。

### 1. リポジトリファイルを作成

手動で `repository/QuickShop.json` を作成します。

### 2. プラグインを管理対象に追加

```bash
/mpm add QuickShop
```

`mpm.json` にプラグイン情報が追加されます。

### 3. インストール

```bash
/mpm install
```

プラグインがダウンロードされ、`plugins/` ディレクトリに配置されます。

### 4. サーバーを再起動

```bash
/restart
```

プラグインが有効化されます。

## 🔒 APIバージョン互換性チェック

プラグインをインストールする前に、MPMはプラグインの `api-version` とサーバーのMinecraftバージョンの互換性を自動的にチェックします。

### 互換性チェックの仕組み

プラグインのJARファイル内の `plugin.yml` または `paper-plugin.yml` に記載されている `api-version` フィールドを読み取り、サーバーのバージョンと比較します。

例えば、サーバーが **1.20** で動作している場合：
- `api-version: 1.20` → ✅ 互換性あり（インストール可能）
- `api-version: 1.19` → ✅ 互換性あり（古いバージョン向けプラグインは互換）
- `api-version: 1.21` → ❌ 互換性なし（サーバーより新しいバージョンを要求）

### 非互換プラグインのインストール

非互換なプラグインをインストールしようとすると、以下のようなエラーメッセージが表示されます：

```
Plugin 'ExamplePlugin' requires api-version 1.21, but the server supports 1.20
--force フラグで強制インストールできます。
```

`--force` フラグを使用することで、互換性チェックを無視してインストールできます：

```bash
# 単一プラグインの強制追加
/mpm add ExamplePlugin --force

# 一括インストールの強制実行
/mpm install --force

# 強制更新
/mpm update --force
```

:::warning
`--force` フラグを使用して非互換なプラグインをインストールした場合、サーバーでエラーが発生する可能性があります。自己責任で使用してください。
:::

## トラブルシューティング

### プラグインがインストールされない

1. **リポジトリファイルを確認**
   ```bash
   # ファイルが存在するか確認
   ls plugins/MinecraftPluginManager/repository/
   ```

2. **mpm.json を確認**
   ```bash
   # プラグインが追加されているか確認
   cat plugins/MinecraftPluginManager/mpm.json
   ```

3. **ログを確認**
   - サーバーログでエラーメッセージを確認してください

### 管理外プラグインが削除される

mpm管理下にないプラグインを保持するには：

```bash
# local/ ディレクトリに移動
mv plugins/YourPlugin.jar plugins/MinecraftPluginManager/local/
```

`local/` ディレクトリ内のプラグインは `mpm removeUnmanaged` コマンドでも削除されません。

### インストール済みプラグインの確認

```bash
# すべてのプラグインを表示（デフォルト）
/mpm list

# すべてのプラグインを表示（明示的）
/mpm list --all

# 管理中のプラグインのみ表示
/mpm list --managed

# 管理外のプラグインのみ表示
/mpm list --unmanaged

# ロック中のプラグインのみ表示
/mpm list --locked
```
